## 1. 第二章目標與流程

**一句話：** 老葛雷委託路德與瑪琳重返北方礦山外圍查明礦工失蹤真相，兩人目睹失蹤礦工已化為骷髏亡靈、擊退被死靈邪氣逼瘋的狂暴洞熊，清開通往洞穴深處的落石，將源頭之謎帶回鎮上——為第三章深入洞穴、對決食人魔揭開序幕。

**玩家動線（承接第一章 `ch1==3`）：**

```
第一章完（ch1==3，緹娜已付哥布林報酬）
   │
   ├─(A) 芳蕾鎮・水井旁 → 和【老葛雷】對話 → 接下委託        [action ch2_take → ch2=1]
   │        緹娜／巴頓等 NPC 台詞同步更新，指引玩家去找老葛雷
   │
   ├─(B) 鎮北出口 → 進入【礦山外圍 Mine 場景】（ch2==1）
   │        正常遇敵：骷髏礦工／獸人挖掘者／礦坑飛魔（沿用 encGroup="mine"）
   │        深處舊礦道口觸發過場 → CUTS "mine_truth"（骷髏礦工＝失蹤死者，真相初揭）
   │
   ├─(C) 舊礦道口的【BearMark】（ch2==1 才顯示）→ 碰觸開打
   │        精英戰：狂暴洞熊 bear_dire（enc="ch2_bear"）
   │        勝利 → [BATTLE_JS 分支：ch2=2、落石鬆動、掉落獎勵]
   │
   ├─(D) 回芳蕾鎮 → 向【老葛雷】回報（ch2==2）             [action ch2_report → ch2=3 + 獎勵]
   │        情緒收束：確認亡者身分、老葛雷托付玩家追查深處源頭
   │
   └─(E) ch2==3：第二章完。洞穴 Cave 深處落石解鎖（銜接第三章食人魔）
            目標 HUD 導向「深入洞穴、追查邪氣源頭」
```

**與既有場景的關係：**
- 「礦山外圍」＝現有 **Mine 場景**（序章教學就在這裡，step==0 用 tutorial 遇敵，step>=3 後自動切 `mine` 組，第二章沿用）。
- 「洞穴深處」＝現有 **Cave 場景**。Cave 現有觸發區 `"深處被落石封住了（第二章開放）"`（`minStep:3`）即為預留鉤子——第二章末把它改成 `ch2>=2` 條件即解鎖（見第 8 節）。
- 第二章**不需要新場景**，只在 Mine 加一個 BearMark 與兩個觸發區、在 Cave 改一個觸發條件。

## 2. 新旗標設計

沿用現有整數狀態機風格（`g_flags` 內的 `step`/`reg`/`ch1` 都是整數，配上 `gotRing`/`gotSword` 這類 0/1 一次性旗標）。`when` 條件語法只支援 `always` 或 `旗標==數字` / `旗標>=數字`（引擎 regex `^(\w+)(==|>=)(\d+)$`，未定義旗標視為 0）——以下設計全部落在這個語法內。

| 旗標 | 型別 | 值域與語意 | 誰設定 |
|------|------|-----------|--------|
| `ch2` | 整數狀態機 | `0`=未接委託（老葛雷可發委託）／`1`=調查中（去礦山外圍）／`2`=已擊退洞熊、深處鬆動（回鎮回報）／`3`=已回報，第二章完、深處解鎖 | `ch2_take`(→1)、洞熊戰勝利(→2)、`ch2_report`(→3) |
| `relic` | 整數狀態機 | `0`=無／`1`=撿到阿吉的頭盔（未上繳）／`2`=已上繳領賞 | 礦山頭盔 pickup(→1)、`relic_turnin`(→2) |
| `mira2` | 整數狀態機 | `0`=未接／`1`=藥草支線進行中／`2`=已完成領賞 | `mira_start`(→1)、`mira_reward`(→2) |
| `herb` | 整數計數器 | `0`~`3`＝已採集鏡草株數 | 東之森 3 個藥草 pickup（各 +1） |

> 現有旗標對照（不改）：`step`(0→3→4 序章)、`reg`(0/1 登錄)、`ch1`(0→1→2→3)、`gotRing/gotSword/gotPotion/gotEar`、`c_*`(過場一次性)、`eqTitle`。
> **ch2 起點的隱含前提是 `ch1==3`**——委託由老葛雷在 `ch1==3` 時發出，接下即 `ch2=1`，其後 `ch1` 恆為 3。所有 ch2 的 DLG 條目排在 `ch1==3` 之前即可正確搶匹配（first-match-wins，見第 3 節排序）。

## 5. 精英戰設計：洞熊 vs 無 boss

### 5-1. 推薦：**採用洞熊精英戰（狂暴洞熊）**

**理由：**
1. **節奏需要高潮。** 第二章若只有「探索＋真相過場」，會是全篇最平的一章；夾在第一章哥布林頭目與第三章食人魔 boss 之間，缺一個張力頂點會讓中段鬆掉。
2. **素材零成本。** `bear`（熊）素材已在庫、設計文件也早把洞熊列為「礦山外圍・精英」。無需 `/gen-art`。
3. **敘事可縫合。** 把洞熊重新定位為「被深處滲出的死靈邪氣逼瘋、佔據崩塌礦道的狂暴洞熊」——牠不再只是野生動物，而是「異變」的具體化身，和骷髏礦工共享同一個病灶。擊退牠＝清開落石＝真相與路徑同時打通，一石三鳥。
4. **教學價值。** 一隻高血、慢速、偶爾全體橫掃的重擊型精英，正好逼玩家用上第一章學到的「防禦（減半）」與探索者的「治癒之風」，把戰鬥系統的深度帶出來。

**「無 boss」方案（備案，不推薦）：** 第二章改為純調查——骷髏群當作稍硬的一般遇敵、以真相過場作結。優點是實作最省、氣氛更懸疑冷冽；缺點是三章連續下來中段無記憶點，且浪費了現成的熊素材。若 John 想走「安靜恐怖」路線可選此案，但建議至少保留一場「骷髏礦工群（3 隻 skeleton）」的腳本遇敵當小高潮。

### 5-2. 數值建議（對齊 pacing.mine ＝ Lv5–8）

**現況：** `bear` 目前是 hp70 / atk16 / def9 / spd5 / exp28 / gold18，作為 `encounters.mine` 裡的一場單體強敵存在——**這組數值當「一般遇敵」剛好，當「章末精英」太軟**（血量只有哥布林頭目 hp300 的 1/4）。

**建議：新增獨立精英怪 `bear_dire`（狂暴洞熊），不要直接覆蓋野生 `bear`。**
理由：野生 `bear` 還在 `encounters.mine` 當隨機強敵用，若把牠改成 boss 級血量，隨機撞上會變勸退。分成兩隻最乾淨。

| 欄位 | 野生 `bear`（維持或微調） | **新增 `bear_dire`（章末精英）** |
|------|------|------|
| name | 洞熊 | 狂暴洞熊 |
| sprite | bear | bear（同素材，戰鬥描邊由 build 自動生成） |
| hp | 70（可微升到 80） | **260** |
| atk | 16（可微升到 18） | **28** |
| def | 9 | **12** |
| spd | 5 | **7** |
